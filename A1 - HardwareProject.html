Hardware Project

By: Praneeth Suryadevara

Scenario

The user clicks on an item they want to purchase and the request gets sent to the application’s server.

Instructions

  1. Mouse
    1. Clicking the mouse creates contact with the electrical components inside the mouse, sending a signal to the bus that data needs to be sent.
    2. The mouse makes a data packet consisting of 3 to 5 bytes that are sent every time the mouse state changes, including being clicked; contains movement in X and Y axis since sending of last packet, and 1 or 0s indicating whether mouse buttons have been clicked.
  2. USB Cable
    1. Assuming that this is a wired mouse, the mouse transmits the data through a cable that is attached to it and towards the computer. The cord will likely be a USB-A connector type.
  3. Ports
    1. This data is then taken to a USB port inside the computer, which allows a cable to be plugged into them and transmit data. These are connected to the motherboard and allow connection to subsequent parts.
  4. Motherboard
    1. The motherboard uses its built-in wires to transmit data to the southbridge where the USB input data can be processed.
  5. Southbridge
    1. Also known as the I/O Controller Hub on Intel devices, this is the general station for handling input and output data. Here, an interrupt signal is also made to be sent to the CPU, informing it to stop and process the mouse inputs.
  6. Data Bus
    1. The data bus allows the information (including the interrupt) to be transmitted away from the southbridge using electrical wires so it can be received by other components on the motherboard.
  7. Northbridge
    1. It is received in the northbridge, which is able to distribute it to other components on the motherboard.
  8. Memory Slots
    1. The memory slots receive the data so it can be sent to the DRAM.
  9. RAM
    1. The input data is all placed into the random access memory which serves as a temporary storage area for the CPU to deal with data more efficiently.
    2. I/O-Mapped I/O: A block in the RAM is set aside for peripherals, and the mouse data is placed there when it has something to send to the CPU, more efficient than the alternative (Memory Mapped I/O)
  10. CPU Socket
    1. This is the resting place for the CPU to be placed in, and contains the series of pins that connect it to the motherboard and allow data to be transmitted to the CPU.
  11. CPU and RAM
    1. After receiving the interrupt signal, the first task is to pause the current task and handle the input.
    2. The CPU might increase its clock rate if it needs to handle a high number of mouse and potentially keyboard inputs.
    3. The Memory Management Unit determines that the data needs to be retrieved from RAM storage rather than ROM.
    4. The Control Unit of the CPU fetches the data first and decodes the instruction into a command that can be easily processed. It uses electrical signals to direct the other parts to do so.
    5. The ALU (Arithmetic Logic Unit) in the CPU receives this data and performs operations on it.
    6. The results of these operations are stored in the CPU’s primary cache so they can be accessed quicker during subsequent operations (however, there aren’t many since it is a single mouse click).
    7. The cycle repeats until all of the data is processed by the CPU.
    8. CPU creates a signal to inform the network card while storing the packets that are to be transmitted in the RAM.
  12. CPU Socket
    1. Data from the memory unit is transferred back to the CPU socket so it can be sent out from the CPU.
  13. Northbridge/Southbridge
    1. The signal is transferred to the northbridge and back to the southbridge, which recognizes as it output to be sent to the PCI slots via the PCI Bus.
  14. PCI Slots
    1. The signal is received and transferred over to the slot which contains the network card.
  15. Network Card
    1. The network card first retrieves and reads the packets that are stored in the RAM.
    2. Network card uses its own CPU/RAM to calculate how to send the data.
    3. The network card sends the data to the application’s server.
